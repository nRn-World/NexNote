# Contributing to NexNote 🚀

First off, thank you for considering contributing to NexNote! It's people like you that make NexNote such a great tool for the developer community.

## 🌈 How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/nRn-World/NexNote/issues) to see if the bug has already been reported.
*   If not, open a new issue. Include a clear title, a description of the problem, and steps to reproduce the bug.

### Suggesting Enhancements
*   Open an issue with the tag "enhancement".
*   Explain the feature you'd like to see and why it would be useful.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure the test suite passes.
4.  Make sure your code lints.
5.  Issue that pull request!

## 💻 Local Development Setup

### Prerequisites
*   Node.js (v18 or higher)
*   npm

### Setup Steps
1.  **Fork and Clone:**
    ```bash
    git clone https://github.com/YOUR-USERNAME/NexNote.git
    cd NexNote
    ```
2.  **Install Dependencies:**
    ```bash
    npm install
    ```
3.  **Environment Variables:**
    *   Create a `.env.local` file.
    *   Copy the keys from `.env.example` and fill in your own Firebase and Gemini API credentials.
4.  **Run Dev Server:**
    ```bash
    npm run dev
    ```

## 🎨 Coding Standards
*   We use **Tailwind CSS** for styling. Please avoid raw CSS where possible.
*   Use **Lucide Icons** for consistency.
*   Follow the existing folder structure in `src/components`.
